How It Works

Three layers of protection

Your agent wants to swap tokens. Here's what happens before any funds move.

STEP 01

Connect

One command adds Aegis to Claude, GPT, or any MCP-compatible agent. Your agent gets four safety tools it can call before every trade.

STEP 02

Scan & Simulate

Is this contract a honeypot? Will this swap get front-run? Aegis checks the contract against 22 exploit patterns, then simulates the trade on a forked chain. No real funds at risk.

STEP 03

Enforce On-Chain

Even if software fails, the blockchain doesn't. AegisGateway verifies a signed safety attestation before executing any DeFi call. The Uniswap v4 hook blocks unsafe swaps directly at the pool.

Integration

Add it in 30 seconds

Aegis is an MCP server. If your agent speaks MCP, it already works.

Terminal
# Add Aegis to Claude Code
$ claude mcp add aegis npx aegis-defi

# Or run directly
$ npx aegis-defi

# Your agent now has 4 safety tools:
#   scan_contract      - Static analysis for 22 exploit patterns
#   simulate_transaction - Forked-chain tx simulation
#   check_token        - Anti-honeypot token checks
#   assess_risk        - All-in-one risk assessment
JSON - claude_desktop_config.json
{
  "mcpServers": {
    "aegis": {
      "command": "npx",
      "args": ["aegis-defi"]
    }
  }
}
TypeScript
import { createAegisServer } from "aegis-defi";

// Create and start the MCP server
const server = createAegisServer();

// Connect via stdio transport
const transport = new StdioServerTransport();
await server.connect(transport);

// The server exposes 4 tools:
//   scan_contract, simulate_transaction,
//   check_token, assess_risk
// assess_risk returns signed attestations for
// ALLOW/WARN decisions (chainId-bound signatures)
On-Chain Enforcement

Two on-chain safety nets

Off-chain checks can be skipped. These can't. Smart contracts on Base enforce safety at the transaction and pool level.

AegisGateway Base

Your agent routes DeFi calls through the Gateway instead of calling contracts directly. The Gateway checks for a valid safety attestation (signed by Aegis after scanning and simulating) before forwarding the call. No valid attestation = transaction reverts. Works for any DeFi action: swaps, lending, staking, approvals.

Network Base (Chain ID 8453)
Security EIP-2 check, ecrecover guard, nonReentrant

AegisSafetyHook Uniswap v4

Installed directly on Uniswap v4 pools. The hook runs beforeSwap on every trade through that pool: it checks if either token is flagged as dangerous and verifies a valid safety attestation. This protects the pool itself, blocking unsafe swaps even from agents that don't use the Gateway.

Type beforeSwap hook, immutable owner
Scope Pool-level -- protects all swaps, not just Aegis users
Action Reverts if token flagged, attestation missing/expired, or risk too high
Real-World Impact

$62M lost. All preventable.

These are real incidents from the last year. Each one would have been blocked by Aegis before any funds moved.

$50M LOST

Aave Swap Disaster

March 2026 -- A whale swapped $50M USDT for AAVE tokens. Routed through a pool with only $73K liquidity. MEV bots extracted $34M.

Aegis catches it: simulate_transaction flags extreme price impact. Agent gets BLOCK decision.
$11M EXPLOIT

Cork Protocol Hack

May 2025 -- Missing access controls in CorkHook.beforeSwap() on Uniswap v4. Attacker bypassed all safety checks.

Aegis catches it: AegisSafetyHook requires valid attestation before any swap executes.
$1.78M DRAINED

Moonwell Oracle Attack

Feb 2026 -- Oracle misconfiguration on Base priced cbETH incorrectly. Attacker exploited the price discrepancy.

Aegis catches it: Pattern 17 (Oracle Manipulation) detects inline reserve-based pricing.
